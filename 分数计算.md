# OSTI 分数计算说明

## 1. 单题分值规则

- 默认规则（适用于大多数 regular 题）：A = 1 分，B = 2 分，C = 3 分。
- 覆盖规则：如果某一道题配置了 scoreMap，那么该题按 scoreMap 计算，不再使用默认分值。

## 2. 第 19 题当前分值

题干：你在教室上课时会选择听老师、看网课、还是打游戏？

- A：听老师，优先跟住课堂节奏。= 3 分
- B：看网课，按自己的节奏补知识。= 2 分
- C：打游戏，先放松一下再说。= 1 分

说明：第 19 题使用了自定义 scoreMap（A: 3, B: 2, C: 1）。

## 3. 系统怎么算分

### 3.1 单题得分

1. 先看题目有没有 scoreMap。
2. 如果有，就用 scoreMap 中该选项对应的分数。
3. 如果没有，就用默认分（A=1，B=2，C=3）。

### 3.2 维度得分

- 每个维度由 2 道 regular 题组成。
- 维度得分 = 这 2 道题的得分之和。
- 例如 Ac1 维度：第 19 题得分 + 第 20 题得分。

### 3.3 维度等级

- 维度分 <= 3：L
- 维度分 = 4：M
- 维度分 >= 5：H

## 4. 关键代码位置

- 题库与分值配置：src/data/osti-data.ts
- 算分入口（默认分 + scoreMap 覆盖）：src/lib/osti-engine.ts 中的 scoreOption
- 维度累计：src/lib/osti-engine.ts 中的 computeDimensions
- 等级转换：src/lib/osti-engine.ts 中的 scoreToLevel

## 5. 日后如何维护和修改

1. 只改题目文案（不改分数）
- 在 QUESTION_BANK 对应题目的 prompt 或 options.text 中修改文案即可。
- 不增加 scoreMap 时，分数规则保持默认。

2. 修改某一道题的分数（推荐做法）
- 在该题对象里添加或修改 scoreMap。
- 示例：

```ts
scoreMap: {
  A: 3,
  B: 2,
  C: 1,
}
```

3. 修改全局默认分值（谨慎）
- 修改 SCORING_CONFIG.defaultScoreMap。
- 这会影响所有“没有 scoreMap 的 regular 题”。

4. 新增或维护反向题
- 反向题通常设置为 A=3, B=2, C=1。
- 现有反向题可参考第 14 题和第 27 题的写法。

5. 修改后的验证建议
- 运行构建：npm run build
- 抽测至少 3 组答案：
  - 全选 A
  - 全选 B
  - 全选 C
- 对比 dimensionScores 和 dimensions 是否符合预期。
